## 前端组件

### FE-01: 公共布局组件 (Layout)
- **职责**: 页面整体布局框架，包含导航栏、页脚、语言切换
- **技术**: Next.js App Router Layout
- **接口**: 语言切换回调、用户登录状态上下文

### FE-02: 非遗展示模块 (Heritage Display)
- **职责**: 非遗项目列表页、详情页、分类浏览、图片画廊、视频播放
- **技术**: React 组件 + Next.js SSG/ISR
- **接口**: 调用后端 Heritage API、Media API

### FE-03: 用户模块 (User Module)
- **职责**: 注册、登录、密码重置、个人中心、收藏夹
- **技术**: React 组件 + React Context（认证状态）
- **接口**: 调用后端 Accounts API

### FE-04: 评价模块 (Review Module)
- **职责**: 评分组件、评论列表、评论表单、图片上传
- **技术**: React 组件
- **接口**: 调用后端 Reviews API、Media API（文件上传）

### FE-05: 搜索模块 (Search Module)
- **职责**: 搜索框、搜索结果页、分类筛选器
- **技术**: React 组件 + 防抖搜索
- **接口**: 调用后端 Search API

### FE-06: 社区论坛模块 (Forum Module) `[P1]`
- **职责**: 论坛分区列表、帖子列表、帖子详情、发帖/回帖
- **技术**: React 组件
- **接口**: 调用后端 Forum API

### FE-07: 国际化模块 (i18n)
- **职责**: 中英双语文本管理、语言切换、URL 路由国际化
- **技术**: next-intl 或 next-i18next

## 后端组件（Django Apps）

### BE-01: accounts（用户认证模块）
- **职责**: 用户注册、登录、密码重置、邮箱验证、RBAC 权限管理
- **模型**: User, Role, Permission
- **对外接口**: REST API（注册、登录、Token 刷新、用户信息 CRUD）
- **模块边界**: 不直接访问其他模块的数据库表

### BE-02: heritage（非遗项目模块）
- **职责**: 非遗项目 CRUD、分类管理、状态管理（草稿/发布/下架）
- **模型**: HeritageItem, Category, Region, Inheritor
- **对外接口**: REST API（项目列表、详情、创建、编辑、状态变更）
- **模块边界**: 通过 accounts 模块验证权限，通过 media 模块管理附件

### BE-03: reviews（评价模块）
- **职责**: 评分、评论 CRUD、评论管理
- **模型**: Review, Rating
- **对外接口**: REST API（评论列表、创建、删除、评分统计）
- **模块边界**: 依赖 accounts（用户信息）和 heritage（项目关联）

### BE-04: forum（社区论坛模块）`[P1]`
- **职责**: 论坛分区、帖子 CRUD、回复管理
- **模型**: ForumSection, Post, Reply
- **对外接口**: REST API（分区列表、帖子 CRUD、回复 CRUD）
- **模块边界**: 依赖 accounts（用户信息）

### BE-05: media（媒体管理模块）
- **职责**: 文件上传处理、媒体元数据管理、文件类型/大小校验、本地文件存储管理
- **模型**: MediaFile
- **对外接口**: REST API（文件上传、媒体元数据 CRUD）
- **模块边界**: 使用 Django FileSystemStorage 存储文件到本地磁盘（通过 Django STORAGES 配置，未来可切换为 S3），不直接访问其他模块数据

### BE-06: search（搜索模块）
- **职责**: 全文搜索、分类筛选、搜索索引同步
- **模型**: 无独立模型（使用 Elasticsearch 索引）
- **对外接口**: REST API（搜索查询、筛选）
- **模块边界**: 从 heritage 模块同步数据到 Elasticsearch

### BE-07: core（核心公共模块）
- **职责**: 公共工具函数、分页器、权限类、异常处理、中间件
- **模型**: 无
- **对外接口**: 内部 Python 模块，供其他 app 引用

## 数据层组件

### DL-01: PostgreSQL（主数据库）
- **职责**: 存储结构化数据（用户、非遗项目、评价、论坛帖子）
- **使用方**: 所有后端 Django Apps（通过 Django ORM）

### DL-02: Elasticsearch（搜索引擎）
- **职责**: 全文搜索索引、中英文分词、搜索结果排序
- **使用方**: search 模块
- **数据来源**: heritage 模块数据同步

### DL-03: 本地文件存储（Django FileSystemStorage）
- **职责**: 图片、视频、音频文件存储，通过 Django 开发服务器提供访问
- **使用方**: media 模块（文件存储和管理）、前端（通过后端 URL 访问媒体文件）
- **扩展性**: 通过 Django STORAGES 配置，未来可无缝切换为 AWS S3 + CloudFront

### DL-04: Redis（缓存）
- **职责**: API 响应缓存、会话存储、速率限制计数器
- **使用方**: 所有后端模块

# 非遗博物馆 - 需求文档

## 1. 意图分析摘要

- **用户请求**: 开发一个面向全球用户的中国非物质文化遗产博物馆 Web 平台，提供非遗项目的维护、展示、评价、交流功能
- **请求类型**: New Project（全新项目）
- **范围估计**: System-wide（多组件系统）
- **复杂度估计**: Moderate（中等复杂度，MVP 策略降低初期复杂度）
- **开发策略**: MVP 优先，核心功能先行，高级功能（虚拟展厅/3D、智能推荐）后续迭代

## 4. 技术决策

| 决策项 | 选择 | 说明 |
|--------|------|------|
| 前端框架 | React / Next.js | SSR/SSG 支持，SEO 友好，i18n 生态成熟 |
| 后端框架 | Python / Django + DRF | 成熟稳定，ORM 强大，REST API 开发高效 |
| 主数据库 | PostgreSQL | 关系型数据，结构化非遗项目数据 |
| 搜索引擎 | Elasticsearch | 全文搜索，中英文分词支持 |
| 文档数据库 | MongoDB | 灵活存储非结构化内容（评论、论坛帖子等） |
| 对象存储 | 本地文件系统 (Django FileSystemStorage) | 开发阶段本地存储，通过 STORAGES 配置未来可切换 S3 |
| CDN | 无（本地开发不需要） | 未来部署时可接入 CloudFront |
| 部署平台 | 本地 Docker Compose | PostgreSQL + Elasticsearch + Redis 容器化运行 |
| 语言 | TypeScript (前端) + Python (后端) | 类型安全 + 开发效率 |

## 5. MVP 范围定义

### MVP（第一阶段）包含：
- 用户认证（邮箱+密码，RBAC）
- 非遗项目 CRUD + 多媒体管理
- 图文视频展示
- 全文搜索 + 分类筛选
- 星级评分 + 文字评论
- 评论区
- 用户个人中心（资料、收藏）
- 后台管理基础功能
- 中英双语界面
- AWS 部署 + CDN
+ 本地 Docker Compose 部署

### 后续迭代包含：
- 社区论坛
- 图片评论
- 智能推荐
- 虚拟展厅/3D 展示
- 浏览历史
- 数据统计仪表板
- 社交账号登录扩展

## 6. 约束与假设

### 约束
- 初期预算有限，采用 MVP 策略
- 初期数据量小（<100 非遗项目）
- 团队规模较小

### 假设
- 非遗项目内容由内容管理员手动录入
- 多媒体素材由内容管理员提供
- 初期不需要自动化内容采集
- 用户增长为渐进式
